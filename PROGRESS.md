## Decisions log

- **2026-07-03 — Embedding-based dedup over pg_trgm/simhash**: different outlets paraphrase headlines for the same story, which text-similarity misses; `gemini-embedding-001` is free and pgvector is built into Supabase, so semantic matching costs nothing extra. Brute-force nearest-group query (no ANN index) because volume is a few hundred articles/day.
- **2026-07-03 — Summarize per group, not per article**: the whole point of grouping is one entry per story; also cuts LLM calls (helps free-tier quota).
- **2026-07-03 — Fallback order 3.5-flash → 3.1-flash-lite → 2.5-flash-lite**: Gemini Pro models left the free tier in April 2026, so the chain is Flash-only, ordered quality-first with the highest-RPD model last.
- **2026-07-03 — Two Edge Functions (`ingest`, `summarize`), not one**: decouples fetching from LLM quota; the 15-min `summarize` pass doubles as the retry pass (no third function needed) since transiently-failed rows simply stay `pending`.
- **2026-07-03 — Cleanup as pure SQL in pg_cron**: a single DELETE with cascade needs no Edge Function.
- **2026-07-03 — Articles/groups/summaries are global, not per-user**: 3–4 trusted users; per-user filtering happens client-side via `user_sources`. Avoids duplicating fetch/summarize work per user.
